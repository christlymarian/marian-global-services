Security Disclosure

This page explains our high-level security practices and how to report vulnerabilities or security concerns.

Our security practices

  • Data in transit: Site forms and uploads use TLS (HTTPS) where available. For local testing, we use secure transport to SMTP and other services.
  • Data at rest: Uploaded files are stored temporarily, processed for quote evaluation, and removed automatically from server storage unless otherwise agreed.
  • Access control: Administrative access to production systems is restricted to authorized personnel and uses secure authentication methods.
  • Backups & monitoring: Regular backups and logs are retained to support recovery and incident response.

Reporting vulnerabilities

If you discover a potential security issue affecting our website or services, please report it responsibly.

Email: security@marianglobalservices.com

When reporting, please include:

  • A clear description of the issue and steps to reproduce it.
  • Any relevant URLs, screenshots, timestamps, and test accounts (if applicable).
  • Your contact details so we can follow up.

Coordinated disclosure

We aim to acknowledge reports promptly and work with reporters to verify and remediate issues. We ask that you do not publicly disclose the issue until we have had a reasonable opportunity to address it.

Limitations

This page provides a summary of security practices. For production deployments and compliance (e.g., ISO, SOC2), additional controls and audits are recommended.
